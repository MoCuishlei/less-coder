# WORKLOG（阶段工作记录）

用于沉淀项目阶段性记录，按层级归档：

- `L0`：项目基线、协议评审、架构决策
- `L1`：Java 主链路打通、CLI 与打包落地
- `L2`：多语言扩展（Go/JS/TS/C/C++）
- `L3`：稳定性、性能、测试与发布准备
- `L4`：产品化与生态接入（Desktop/IDE/外部集成）

## 记录规范

1. 每次阶段推进至少新增一条记录。
2. 文件命名建议：`YYYY-MM-DD_主题.md`。
3. 每条记录至少包含：目标、变更、验证结果、风险、下一步。
4. 关键决策必须在对应层级补充决策记录（ADR 类内容）。

## 模板与索引

- 每个层级目录下提供 `TEMPLATE.md`。
- 建议每个层级维护本层 `README.md` 索引（可选但推荐）。
