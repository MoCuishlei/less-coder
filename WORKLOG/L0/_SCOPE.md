# WORKLOG（阶段工作留存）

用于沉淀项目阶段性记录，按层级归档：

- `L0`：项目基线与架构决策
- `L1`：MVP 核心能力（Java + LangGraph + CLI）
- `L2`：多语言扩展（Go/JS/TS/C/C++ 等）
- `L3`：稳定性、性能、测试与发布准备
- `L4`：产品化与生态接入（Desktop/IDE/外部 MCP）

## 使用规则

1. 每次阶段性推进至少新增一条记录。
2. 文件命名建议：`YYYY-MM-DD_主题.md`
3. 一条记录至少包含：目标、变更、验证结果、风险、下一步。
4. 关键决策必须在 `L0` 或对应阶段补“决策记录（ADR）”。

## 推荐模板

每个层级目录下放 `TEMPLATE.md`，新记录可直接复制。
